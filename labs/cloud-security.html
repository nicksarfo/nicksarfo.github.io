Project Overview

This lab focused on working with Amazon Web Services in a controlled academic environment. The goal was to gain hands-on experience with common cloud services, access controls, storage permissions, serverless computing, database configuration, and cloud security troubleshooting.

Tools and Technologies

AWS AWS CLI S3 Lambda DynamoDB IAM Concepts Cloud9 Python

What I Did

I completed cloud security labs involving AWS storage, serverless functions, database resources, and command-line administration. This included creating and managing S3 buckets, uploading objects, reviewing access control settings, working with AWS CLI commands, building Lambda functions, and interacting with DynamoDB tables.

I also practiced troubleshooting common cloud configuration issues, such as access control behavior, object permissions, and service-specific errors. These tasks helped reinforce the importance of least privilege, secure configuration, clear documentation, and careful validation when working in cloud environments.

Skills Demonstrated

This project demonstrates practical experience with cloud security fundamentals, AWS service configuration, command-line administration, access control review, serverless computing, database interaction, and technical troubleshooting. It also shows my ability to document cloud tasks in a way that connects technical steps to security outcomes.

Challenges and Lessons Learned

One important lesson from this lab was that cloud security depends heavily on correct configuration. A resource can be technically functional while still being insecure if permissions, access controls, or policies are not reviewed carefully. I also learned that troubleshooting cloud errors requires checking both the service configuration and the permissions model behind the action being performed.

This project is relevant to roles such as Cybersecurity Analyst, Cloud Security Analyst, SOC Analyst, Cloud Support Associate, and Security Operations Specialist. It shows hands-on exposure to cloud platforms, secure configuration concepts, access control review, and technical troubleshooting in AWS.
