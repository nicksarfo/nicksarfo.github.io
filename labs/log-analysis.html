Project Overview

This lab focused on collecting, reviewing, and analyzing security logs from multiple systems in a controlled lab environment. The goal was to identify suspicious activity, correlate events across different log sources, and document findings in a professional cybersecurity report format.

Tools and Technologies

Splunk Windows Event Logs Linux Auth Logs Apache Access Logs Python Pandas SSH SMB

What I Did

I worked with Linux, Windows, and web access logs to identify security-relevant events. This included reviewing authentication activity, examining Windows event IDs, analyzing web access patterns, and using scripts to support log collection and correlation. I also documented issues encountered during the lab, including cases where expected data was unavailable or could not be retrieved successfully.

Skills Demonstrated

This lab demonstrates hands-on experience with log analysis, event correlation, security monitoring, troubleshooting, technical documentation, and basic detection engineering concepts. It also shows the ability to explain incomplete or failed results professionally when lab tools or data sources do not behave as expected.

Challenges and Lessons Learned

One key lesson from this lab was that log analysis often requires troubleshooting missing or incomplete data. In a real security operations environment, analysts must be able to validate whether data sources are properly configured, explain gaps in collection, and avoid making unsupported conclusions when evidence is unavailable.

Career Relevance

This project is relevant to roles such as SOC Analyst, Cybersecurity Analyst, Security Operations Specialist, and Incident Response Analyst. It shows practical experience reviewing security logs, correlating events, documenting findings, and communicating technical results clearly.
